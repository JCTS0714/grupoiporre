## Brand & Style

El sistema de diseño está construido sobre tres pilares fundamentales: **Calidad, Profesionalismo y Control**. La identidad visual proyecta una imagen de autoridad moderna y eficiencia tecnológica en el sector de la administración de fincas.

El estilo visual elegido es **Corporate / Modern** con sutiles toques de **Glassmorphism**. Se caracteriza por una estética limpia, una rejilla estructurada y el uso estratégico de degradados que evocan profundidad y tridimensionalidad, inspirados en la arquitectura del logotipo. El objetivo es transmitir una sensación de seguridad y transparencia, donde cada elemento tiene un propósito funcional claro. La interfaz debe sentirse robusta pero accesible, eliminando el desorden visual para enfocarse en la claridad de los datos y la gestión operativa.

## Colors

La paleta se centra en el dinamismo entre el **Azul Vibrante** y el **Violeta**, utilizando el degradado como un recurso visual clave para representar la integración de servicios.

- **Primario (Azul):** Representa la tecnología, el agua (cisternas) y la confianza institucional.
- **Secundario (Violeta):** Aporta un toque de distinción, exclusividad y modernidad.
- **Deep Navy:** Se utiliza para la tipografía principal y elementos de navegación, proporcionando un contraste superior al negro puro.
- **Degradados:** Deben aplicarse en botones de acción principal, encabezados de tarjetas destacadas y elementos iconográficos de gran formato, fluyendo siempre de izquierda (azul) a derecha (violeta).
- **Modo de color:** El sistema está optimizado para fondos claros (`#FFFFFF` o `#F8F9FA`) para mantener la sensación de limpieza y amplitud.

## Typography

La tipografía se divide en dos familias complementarias:
1. **Manrope** para titulares: Su estructura geométrica y moderna refuerza la idea de control y arquitectura. Se utiliza en pesos pesados para establecer una jerarquía clara.
2. **Hanken Grotesk** para cuerpo y etiquetas: Ofrece una legibilidad excepcional en textos densos, como informes financieros o reglamentos internos.

Los textos de cuerpo deben utilizar un color gris muy oscuro (`#1E293B`) en lugar de negro para reducir la fatiga visual. Las etiquetas (labels) se presentan frecuentemente en mayúsculas con un ligero espaciado entre letras para denotar profesionalismo administrativo.

## Layout & Spacing

El sistema utiliza una **rejilla fluida de 12 columnas** para escritorio y una **rejilla de 4 columnas** para dispositivos móviles. El ritmo vertical se basa en una unidad de 8px para garantizar consistencia.

- **Contenedores:** El contenido principal debe estar contenido en un ancho máximo de 1280px para evitar líneas de texto demasiado largas que dificulten la lectura.
- **Densidad:** El espaciado es generoso (espacios en blanco) para transmitir orden. Los informes y tablas de gestión pueden reducir el espaciado interno (padding) para permitir la visualización de más datos sin necesidad de scroll excesivo.
- **Adaptabilidad:** En móviles, los elementos de la cuadrícula se apilan verticalmente, y los márgenes laterales se reducen a 16px para maximizar el área de contenido.

## Elevation & Depth

La jerarquía visual se establece mediante **Capas Tonales** y **Sombras Ambientales**.

1. **Nivel 0 (Fondo):** Superficie neutra clara (`#F8F9FA`).
2. **Nivel 1 (Tarjetas/Contenedores):** Fondo blanco puro con una sombra muy suave y difusa (Blur: 16px, Opacity: 4%, Color: Navy). Esto crea una separación sutil del fondo.
3. **Nivel 2 (Modales/Dropdowns):** Sombras más definidas y bordes nítidos de 1px en un tono gris muy claro para enfatizar la flotación sobre el contenido.
4. **Interacción:** Los elementos interactivos pueden usar un efecto de "Glassmorphism" suave (desenfoque de fondo de 8px) cuando se superponen a imágenes o degradados, manteniendo la legibilidad sin sacrificar la estética moderna.

## Shapes

El sistema adopta un lenguaje de formas **Rounded (Redondeado)**. 

Este nivel de redondez (0.5rem / 8px) suaviza la rigidez de los datos financieros y administrativos, haciendo que la plataforma se sienta acogedora y humana. 
- Los botones principales y las tarjetas utilizan el radio estándar de 8px.
- Los elementos de estado (chips) y avatares utilizan radios de tipo "Pill-shaped" para distinguirse claramente de los contenedores de información estructural.
- Los campos de entrada (inputs) mantienen el radio de 8px para consistencia visual con el sistema de botones.

## Components

### Botones
- **Primario:** Degradado de azul a violeta, texto blanco en negrita. Efecto de elevación al pasar el cursor (hover).
- **Secundario:** Borde de 1.5px en azul con fondo transparente o blanco.
- **Terciario (Ghost):** Sin borde, solo texto azul, para acciones menos críticas como "Cancelar".

### Cards (Tarjetas de Servicio)
Contenedores blancos con bordes suaves. Deben incluir un icono en la esquina superior izquierda sobre un fondo circular con el color de la categoría (ej. verde para jardines, rojo para seguridad).

### Entradas de Datos (Inputs)
Fondo blanco, borde gris claro de 1px. Al recibir el foco (focus), el borde cambia a azul sólido con un resplandor suave. Las etiquetas de los campos deben estar siempre visibles arriba del input.

### Chips de Estado
- **Pagado/Activo:** Fondo verde claro, texto verde oscuro.
- **Pendiente/Mantenimiento:** Fondo amarillo claro, texto ocre.
- **Mora/Urgente:** Fondo rojo claro, texto rojo oscuro.

### Listas de Gestión
Filas con alternancia de color de fondo (zebra-striping) muy sutil para facilitar la lectura de registros financieros. Los encabezados de columna siempre deben estar fijos (sticky) en tablas largas.
